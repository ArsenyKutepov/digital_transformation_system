Интеллектуальная система поддержки принятия решений по цифровой трансформации организационного объекта
Описание системы
Интеллектуальная система поддержки принятия решений (СППР) предназначена для оценки цифровой зрелости организаций и адаптивного планирования цифровой трансформации. Система использует нечёткую логику Мамдани, включающую базу из двухсот пятидесяти пяти правил, для оценки текущего состояния организации и алгоритм Deep Q-Network для генерации оптимальной дорожной карты трансформации с учётом организационной инерции и когнитивной нагрузки персонала.

Основные возможности системы включают оценку цифровой зрелости по двадцати пяти показателям, сгруппированным в девять блоков; адаптивное планирование цифровой трансформации на основе DQN; управление спринтами для пошаговой реализации плана; расширенную аналитику с прогнозированием и бенчмарками; ролевую модель с разграничением прав администратора, аналитика и наблюдателя; REST API для интеграции с внешними системами; а также экспорт отчётов в форматах Excel, JSON, CSV и HTML.

Необходимое программное обеспечение
Для работы системы требуется Python версии 3.10 или выше. База данных SQLite встроена в Python и не требует отдельной установки. Для доступа к веб-интерфейсу необходим любой современный браузер Chrome, Edge или Firefox.

Установка и запуск
Первым шагом необходимо клонировать репозиторий или скачать исходные файлы проекта. После этого следует перейти в корневую папку проекта и создать виртуальное окружение. В операционной системе Windows команда будет следующей: python -m venv .venv, после чего нужно активировать окружение командой .venv\Scripts\activate. В Linux или macOS команды будут соответственно python3 -m venv .venv и source .venv/bin/activate.

Затем необходимо установить все зависимости, указанные в файле requirements.txt, командой pip install -r requirements.txt.

После установки зависимостей система запускается командой python main.py. При успешном запуске в консоли появится сообщение о том, что сервер запущен на адресе http://127.0.0.1:8080.

Для доступа к системе откройте браузер и перейдите по адресу http://127.0.0.1:8080. Для входа в систему可以使用 демо-доступ: логин admin, пароль admin123.

Структура проекта
Проект организован следующим образом. Файл main.py является точкой входа в приложение. Файл config.py содержит все конфигурационные параметры. Папка database содержит модули для работы с базой данных SQLite. Папка core содержит ядро системы, включая реализацию нечёткой логики, DQN-агента, нейронной сети, среды MDP и динамической модели. Папка web содержит веб-сервер на чистых сокетах, маршрутизатор, шаблонизатор и все обработчики HTTP-запросов. Папка api содержит реализацию REST API. Папка export отвечает за генерацию отчётов. Папка scheduler содержит планировщик периодических задач. Папка templates содержит HTML-шаблоны для всех страниц. Папка models предназначена для сохранения обученных моделей DQN. Папки logs, reports и backups предназначены соответственно для хранения логов, сгенерированных отчётов и резервных копий базы данных. Папка data содержит файл базы данных SQLite.

Возможные проблемы и решения
Если при запуске возникает ошибка ModuleNotFoundError, необходимо установить недостающую библиотеку командой pip install с указанием имени библиотеки. Если порт 8080 уже занят другим приложением, следует изменить значение PORT в файле config.py на любой свободный порт, например 8081. При появлении ошибок кодировки в консоли Windows рекомендуется перед запуском выполнить команду chcp 65001 для переключения кодовой страницы на UTF-8. Если база данных не создаётся автоматически, необходимо проверить наличие прав на запись в папке data.
